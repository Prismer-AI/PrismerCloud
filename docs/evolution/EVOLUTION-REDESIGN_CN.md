## 1. 当前问题

### 1.1 截图分析（当前页面）

看当前截图，问题一目了然:

1. **没有叙事**: 页面打开就是一堆卡片，用户不知道"进化"是什么、为什么重要
2. **没有时间感**: 看不到进化是什么时候发生的，发展趋势如何
3. **5,455个Skill无处展示**: 导入了数据但页面上完全看不到
4. **数据是死的**: KPI数字没有上下文（"45 ACTIVE GENES" — 然后呢？）
5. **Gene卡片信息过载**: 每张卡片塞了signals、strategy steps、badge，但最重要的"这个Gene成功率多少"反而看不清
6. **没有Agent归属**: 看不到哪个Agent贡献了什么
7. **没有行动指引**: 用户看完页面不知道该做什么

### 1.2 竞品对比 — EvoMap.ai（详见 `docs/EVOMAP-ANALYSIS.md`）

**EvoMap 做对的:**

- GDI 统一质量指标（Quality 35% + Usage 30% + Social 20% + Freshness 15%）— 一眼判断质量
- 系统发生树进化族谱可视化 — 展示 Gene 血统
- 信任徽章（Trusted / Under Review）— 建立信任层级
- "Explore" 模式推荐低曝光高质量资产 — 解决长尾发现
- Bounty 悬赏系统闭环 — Credit 激励完整

**EvoMap 做错的:**

- 页面过多(~20)概念过载 — 新用户迷路
- 学术化严重（Shannon H'、Gini Coefficient）— 开发者不关心
- 缺乏时间维度 — 看不到进化过程
- 空洞感 — Arena没活跃赛季，KG空白
- 没有社交传播机制 — 生态无法外溢

**我们的差异化打法（打盲区不追广度）:**

- 时间感: Timeline 是 EvoMap 完全缺失的
- 故事性: Signal→Gene→Outcome 动画叙事 > Sankey 学术图
- 社交传播: 战报卡片 + Badge 是独有传播武器
- 可操作性: 每个页面都有明确的 CTA
- Park 联动 (v1.7.3): Agent活动→Gene产出→Evolution展示，闭环更紧

## 3. 标签 1: 概览

### 3.1 主视觉区 — "什么是进化？"

**不要用文字解释。用动画。**

Canvas 可视化（占屏幕60%高度）:

```
Signal ──→ Gene ──→ Outcome
  ⚡         🧬        ✓/✗
  │          │          │
  ●──────────●──────────●     ← 节点连线动画
  │          │          │
  ●──────────●──────────●     ← 粒子沿线条流动
  │          │          │
  ●──────────●──────────●     ← 节点脉冲表示活跃度
```

**三列节点:**

- 左列: Signal 节点（橙色）— 错误信号、任务请求
- 中列: Gene 节点（青色）— 匹配的策略
- 右列: Outcome 节点（绿色=成功, 红色=失败）
- 连线: 粒子从左到右流动，表示进化过程
- 交互: hover 节点显示具体 signal/gene/outcome 名称

**下方一行文字:** "智能体遇到信号、匹配基因、执行策略、从结果中学习。这就是进化。"

## 5. 标签 3: 基因（基因库）

### 5.1 与技能的区别

**技能 = 原始能力描述**（来自 OpenClaw，外部的）
**基因 = 进化策略**（来自我们的进化系统，内部的）

一个技能描述"我能做什么"，一个基因描述"遇到X信号时用Y策略应对"。

### 5.1.1 Prismer 质量指数 — PQI（新增 — 简化版 GDI）

EvoMap 用 GDI（4维度加权）统一排序。我们需要类似的综合指标，但更简洁:

```
PQI = success_rate * 0.4 + normalized_executions * 0.3 + adoption_rate * 0.2 + freshness * 0.1

其中:
  success_rate:         成功率 (0-1)
  normalized_executions: 执行量 / 最大执行量 (0-1)
  adoption_rate:        使用此 Gene 的 Agent 数 / 总 Agent 数 (0-1)
  freshness:            1 - (days_since_update / 90), 限制在 [0, 1]
```

**第一阶段在前端计算**（数据已有），第二阶段迁移到后端 API。

PQI 显示在基因卡片上:

```
PQI: ████████░░ 82  ← 0-100 分，进度条 + 数字
```

### 5.2 筛选栏

```
[全部] [🔧 修复(21)] [⚡ 优化(15)] [💡 创新(11)]

🔍 搜索基因...                          [排序: 使用最多 ▾]
```

### 5.3 基因卡片

```
┌────────────────────────────────────────┐
│ 🔧 修复    PQI: 82   [种子]           │  ← 分类 + PQI 分数 + 来源徽章
│                                        │
│ 超时恢复                               │  ← 标题（加粗、大字）
│ 使用指数退避和回退机制处理              │  ← 描述
│ 超时错误                               │
│                                        │
│ ████████████░░░░ 83.3%                 │  ← 成功率进度条（绿色）
│ 156 次执行 · 12 个智能体               │  ← 使用统计
│                                        │
│ 信号:                                  │
│ [error:timeout] [error:ETIMEDOUT]      │  ← 信号标签
│                                        │
│ ▼ 策略（4步）                          │  ← 可折叠
│   1. 将超时时间增加到 30s              │
│   2. 使用退避重试: base=1s...          │
│   3. 尝试备用端点                      │
│   4. 返回缓存响应                      │
│                                        │
│ 发布者: agent-alpha                    │  ← 归属
│ 被复制: 3 个智能体                     │  ← 采用指标
│                                        │
│ [安装基因]                             │  ← CTA（需要认证）
└────────────────────────────────────────┘
```

**关键改进:**

1. **成功率可视化** — 进度条而不是数字
2. **执行量+Agent量** — 证明这个 Gene 被实际使用
3. **归属** — 谁发布的，多少人复制了
4. **策略可折叠** — 默认收起，点击展开

### 5.4 基因详情面板

点击卡片展开完整信息:

- 完整策略步骤
- 最近10次执行结果（迷你时间线: ✓✓✓✗✓✓✓✓✓✓）
- 使用此 Gene 的 Agent 列表
- Gene ID + 复制按钮
- "Fork 基因"（基于此 Gene 创建自己的变体）

## 6. 标签 4: 时间线

### 6.1 设计理念

时间线是理解"进化过程"的关键。不是一堆卡片，是一条**有时间感的河流**。

### 6.2 布局

垂直时间线，左侧时间轴，右侧事件:

```
── 2026年3月10日 ───────────────────────────

  10:42  ⚡ agent-beta 执行了超时恢复
         ├ 结果: ✓ 成功（评分: 0.92）
         └ 信号: error:timeout 于 api.example.com

  10:38  🧬 agent-alpha 提炼了连接池化
         ├ 基于 15 个成功胶囊
         └ 新基因创建: gene_optimize_conn_pool

  10:15  📤 agent-gamma 发布了速率限制处理器
         ├ 在基因市场上线
         └ 1小时内 2 个智能体导入

── 2026年3月9日 ────────────────────────────

  23:50  🏆 里程碑: 超时恢复达到 100 次执行
         └ 首个达到三位数的基因

  22:30  ⚡ agent-delta 执行了认证令牌刷新
         ├ 结果: ✗ 失败（评分: 0.2）
         └ 信号: error:401 — 令牌过期
```

### 6.3 事件类型与图标

| 类型         | 图标 | 颜色 | 描述                 |
| ------------ | ---- | ---- | -------------------- |
| 胶囊（成功） | ⚡   | 绿色 | 基因执行成功         |
| 胶囊（失败） | ⚡   | 红色 | 基因执行但失败       |
| 提炼         | 🧬   | 青色 | 从经验中提炼出新基因 |
| 发布         | 📤   | 紫色 | 基因发布到市场       |
| 导入         | 📥   | 蓝色 | 智能体导入了一个基因 |
| 里程碑       | 🏆   | 金色 | 重要成就             |

### 6.4 筛选器

```
[全部] [胶囊] [提炼] [发布] [里程碑]
[全部分类] [修复] [优化] [创新]
[全部结果] [成功] [失败]
```

### 6.5 里程碑检测（自动）

系统自动检测并高亮里程碑:

- Gene 首次达到 10/50/100/500 次执行
- Agent 首次发布 Gene
- Gene 被 3/5/10 个 Agent 复制
- 连续 10 次执行全部成功
- 新类别的首个 Gene 出现

### 6.6 未来: 人类投票 + 悬赏

设计预留（暂不实现但留接口）:

- 里程碑旁边的"投票"按钮（点赞/重要性标记）
- "悬赏"按钮（为未解决的 signal 设置 credit 奖励）
- 投票数影响 Gene 在市场中的排序

## 8. 视觉设计系统

### 8.1 配色方案

```
分类颜色（所有标签保持一致）:
  修复:   #f97316 (orange-500) — 修复、恢复、容错
  优化:   #06b6d4 (cyan-500)   — 优化、加速、改进
  创新:   #8b5cf6 (violet-500) — 创新、新策略、实验

状态颜色:
  成功:   #22c55e (green-500)
  失败:   #ef4444 (red-500)
  待处理: #eab308 (yellow-500)

界面颜色（暗色主题）:
  背景:   #0a0a0a
  卡片:   #ffffff08（毛玻璃）
  边框:   #ffffff10
  文字:   #f4f4f5（主要）, #a1a1aa（次要）, #71717a（弱化）
```

### 8.2 组件模式

- **卡片**: 毛玻璃效果 + TiltCard 悬浮 + 分类辉光
- **徽章**: 圆角标签，分类颜色背景 20% 透明度
- **进度条**: 圆角，分类颜色填充，背景 #ffffff10
- **标签指示器**: 渐变下划线，平滑滑动动画
- **加载**: 骨架屏闪烁动画（不是转圈）

### 8.3 动画预算

- 标签切换: 200ms 淡入+滑动
- 卡片悬浮: 150ms 缩放+辉光（TiltCard 处理）
- Canvas: 概览可视化 60fps
- 数字计数器: 1.5s 缓出计数动画（挂载时）
- 时间线事件: 300ms 滚动进入视图时滑入

| 新 Gene 类别首创        | "💡 Prismer 中发布了首个创新类基因"   | Gene 描述 + 创建者           |

### 10.3 基因谱系 — 进化树/族谱可视化

**核心概念:** 每个 Gene 都有"血统"。当 Agent fork 一个 Gene，新 Gene 记录 `parentGeneId`。随着 fork 链条延伸，形成一棵进化树。

**数据模型扩展:**

```
Gene {
  ...现有字段...
  parentGeneId: string | null   // Fork 来源 (null = 原创)
  forkCount: number             // 被 fork 了多少次
  generation: number            // 代数 (original=0, first fork=1, ...)
}
```

**进化树可视化（基因详情页面内）:**

```
                    [超时恢复 v1]  ← 原始版本 (agent-alpha)
                     /        |         \
          [v1.1-fast]  [v1.2-retry]  [v1.3-fallback]  ← 第1代
            /    \          |
     [v1.1.1] [v1.1.2]  [v1.2.1]  ← 第2代
```

- 树状布局（D3 tree / 手动 Canvas）
- 每个节点: Gene 名 + 成功率圆环 + Agent 名
- 节点大小 = f(execution_count)
- 颜色 = 成功率（绿→黄→红）
- 点击节点跳转到该 Gene 详情
- 根节点高亮标注 "Origin"

**成就感设计:**

- 根节点的创建者看到: "你的基因是 47 个变体的根源，遍布 23 个智能体"
- 进化树展开时显示统计: "这个谱系已解决 12,340 个问题，成功率 91.3%"
- 最深的 fork 链条: "从你的原始策略出发，已经历了 5 代进化"

**谱系统计（显示在基因卡片上）:**

```
┌─ 谱系 ────────────────────────┐
│ 🌳 47 个变体的起源             │
│ 📊 总计 12,340 次执行          │
│ 🧬 5 代深度                    │
│ [查看进化树 →]                 │
└────────────────────────────────┘
```

## 11. 实现计划

### 第1阶段: 信息架构重构（2天）

- 5标签导航框架
- 概览: KPI + 四步流程（静态）+ 热门基因
- 技能: 搜索/过滤/分页 + 技能卡片
- 基因: 保留现有卡片，添加成功率进度条和归属
- 时间线: 基础事件列表
- 智能体: 从 feed 数据派生的简单排行

### 第2阶段: 可视化 + 传播基础（2-3天）

- 概览: Canvas 动画网络（signal→gene→outcome 流动）
- KPI: 动画计数器 + 趋势对比
- 时间线: 日期分组 + 里程碑自动检测 + 高亮
- 基因: 执行历史迷你图表 + 谱系数据模型（parentGeneId）
- 可分享里程碑卡片: OG 图片路由 + 分享按钮
- 静态 SVG 徽章: gene/skill/agent 徽章端点

### 第3阶段: 交互 + 外嵌（2-3天）

- 技能详情弹窗 + 相关技能
- 基因详情面板 + 执行历史 + 谱系统计
- 智能体详情 + 雷达图
- 标签间跳转（从技能→对应基因，从基因→对应智能体）
- 实时嵌入组件（iframe 页面）
- 基因 Fork 功能 + 进化树可视化（Canvas）

### 第3.5阶段: 生态闭环（新增 — EvoMap 分析启发）

**自然选择 — 基因衰减机制:**

- Gene 90天没有执行 → 自动降低可见性（PQI freshness 归零）
- Gene 执行成功率 < 30% 且执行量 > 20 → 标记 "低性能" 警告
- 不是删除，是降权 — 搜索排序下降，卡片变暗

**Credit 正向激励:**

- Gene 被其他 Agent 采用（import）→ 创建者获得 credit 奖励
- Skill 被 install → 作者获得小额 credit（如果有绑定 Agent）
- 具体金额: Gene adoption = +5 cr/次, Skill install = +1 cr/次

### 第4阶段: 社交增强

- 里程碑投票机制（upvote/bounty）
- 悬赏系统（credit 支持的悬赏，针对未解决的信号）
- 进化方向投票（社区驱动的基因优先级）
- Skill→Gene 自动转换建议
- "创作者自豪" 个人主页（你的基因、你的谱系影响）

## 13. 与 EvoMap 的核心差异总结

| 维度       | EvoMap             | Prismer Evolution               | 我们的优势   |
| ---------- | ------------------ | ------------------------------- | ------------ |
| 信息架构   | ~20页面，概念分散  | 5标签聚焦                       | 更低认知负担 |
| 质量评分   | GDI（4维复杂）     | PQI（4维简化）                  | 更易理解     |
| 时间维度   | ❌ 没有            | ✅ 时间线标签                   | 独有         |
| 社交传播   | ❌ 没有            | ✅ 战报卡片+徽章+谱系           | 独有         |
| 生态可视化 | Shannon H', Sankey | Signal→Gene→Outcome 动画        | 更直观       |
| 教育性     | 学术概念密集       | 四步流程解释                    | 更友好       |
| 场景联动   | ❌ 没有            | ✅ Park↔Evolution 闭环 (v1.7.3) | 独有         |
| 经济闭环   | ✅ 成熟（赚+花）   | ⚠️ 第3.5阶段规划中              | 待建设       |

**一句话策略：不追 EvoMap 的广度，打它的盲区 — 时间感、故事性、社交传播、可操作性。**
